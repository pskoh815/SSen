# 기간분석(시각화 시트) 구현 명세 v2

원본: `SSen분석.xlsx`의 `시각화` 시트 (B1 시작일 / B2 종료일 입력 → LET/LAMBDA 동적 배열).
**이 시트는 ingest 대상이 아님** — Excel→Parquet 변환 시 skip. API에서 v2 정의로 재구현한다.
참조 구현: `src/ssen/analysis/period_analysis.py` (샘플 파일 검증 완료).
Excel 시트용 교정 수식: `docs/viz_formulas_v2.md` (동일 로직의 LET/LAMBDA 버전).
입력: `start`, `end`. "기간 내" = `start ≤ 날짜 ≤ end`.

## v2 지표 정의

### §1 테마 분석 (`/api/period/themes`)
일별 dedup 후 테마별:
- 누적상승율 = 일별 테마 등락률 복리 [F1,F2]
- 누적거래대금 = 일별 테마 거래대금 합 [F1]
- 상승일비율 = 등락률>0 일수 / 등장일수 (지속성)
- **종합점수 = 정규화(log1p 거래대금)×0.5 + 정규화(누적상승율)×0.3 + 상승일비율×0.2** [F3]
- **로테이션 = 후반 복리수익 − 전반 복리수익** (기간 이등분) — `+` 부상 테마, `−` 소멸 테마
- 상승 TOP10: 종합점수 ↓ / 하락 TOP10: 하락점수 ↓ [F4] / 부상 TOP10: 로테이션 ↓

### §1-T 테마 추이 (`/api/period/theme-trend`)
종합점수 상위 `top`개(기본 20) 테마의 일별 **누적 복리 등락률** 시계열.
일별 dedup 테마 등락률 r에 대해 `cum[i] = (Π(1+r/100) - 1) × 100` (기간 시작부터 누적).
응답: `{ "dates": ["YYYY-MM-DD", ...], "series": [ { "theme1": str, "cum": [float, ...] } ... ] }`
- series 정렬 = 종합점수 내림차순 (프론트 칩 순서·색 고정에 사용)
- 테마가 해당일 미등장이면 직전 누적값 유지(carry-forward) — null 금지
- 프론트: dashboard/index.html `initThemeTrend()` 이미 구현, 이 스키마 그대로 소비
- **참조 구현: `src/ssen/analysis/period_analysis.py`의 `theme_trend(m, top)` — 검증 완료.**
  api/main.py에서는 기존 기간 조회 함수로 5개 컬럼(날짜, 종목코드, 테마(1차),
  테마(1차) 등락률, 테마(1차) 거래대금)만 로드해 이 함수에 넘기면 됨 (파일 내 라우트 예시 주석 참조)
- 검증값(샘플, 2026-02-06~04-29): dates 55개, series 20개(종합점수순),
  통신 cum[-1]=1293.11 / 반도체 288.06 / 우주항공 286.58, 응답 ~9KB, null 없음
- **기간 반영 의무**: 모든 /api/period/* 엔드포인트는 start/end 쿼리 파라미터를 반드시 WHERE 절에
  적용해야 함. 프론트는 기간별로 다른 URL을 호출하므로(클라이언트 캐시 키=URL) 서버가 파라미터를
  무시하면 기간을 바꿔도 화면이 변하지 않는 증상이 나타남. DoD: 서로 다른 기간 2개 호출 결과가
  달라야 한다는 자동 테스트 포함

### §2 강세 주도종목 (`/api/period/leaders`)
선정 조건은 v1 유지: `MAX(기여점수) ≥ 7` AND `최근 3등장일 평균 ≥ 3` AND `순위모멘텀 > 0`, 최소 3등장일.
지표 교정: 순위모멘텀 백분위 기반 [F5], 평균기여점수·등장일수·복리수익률·커버리지·복합점수 [F2,F6],
**첫주도신호일**(첫 기여점수≥7 날짜 — 진입 타이밍 백테스트용). 복합점수 ↓ TOP30.

### §3 강세 분포 (`/api/period/breadth`)
v1 그대로: 기여점수 ≥4 주도 / 1≤x<4 강세 / −1≤x<1 중립 / <−1 약세, 강세비율 = (주도+강세)/전체.
축: 규모(대/중/소) + 시장구분(KOSPI/KOSDAQ).

### §4 강세주도 이벤트 (`/api/period/leader-events`)
v1 그대로: 기여점수 ≥7 행 전체, 종목 누적기여 ↓ → 종목명 ↑ → 날짜 ↑.

## API 설계

기간 조합이 무한 → derived 테이블이 아닌 **on-demand DuckDB 쿼리** (predicate pushdown, ≤300ms).
캐시 키 `(endpoint, start, end, dataset_version)`.
선택 파라미터: `compat=v1` — Excel 원본 수식과 동일 값 반환(대조·검증용).

## 대시보드 탭 "기간분석"

1. **테마 패널**: 상승/하락 TOP10 트리맵(크기=거래대금, 색=상승/하락폭) + **테마 추이**
   (글로우 칩으로 최대 5개 테마 선택 → 누적 복리 등락률 라인 비교, 기본 선택=**기간 내 누적 상승률 상위 5개**)
   ← 핵심 인사이트 뷰. 기울기 둔화/역전이 곧 로테이션 신호 (사분면 산점도 대체, 2026-06 결정)
2. **주도종목 테이블**: 복합점수 정렬, 커버리지 게이지, 첫주도신호일 표시.
   행 클릭 → 해당 종목 기간 내 기여점수 시계열 미니차트
3. **강세 분포**: 규모별·시장별 4색 스택 막대 — 소형주 쏠림(예: 0.70 vs 대형주 0.27) 시각화로
   장세 성격(투기성/주도성) 판단 보조
4. **이벤트 타임라인**: 기여점수≥7 발생을 날짜축에 테마 색으로 표시 → 테마 점화 시점 파악
